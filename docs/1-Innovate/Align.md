## Vision
This is a living document that is subject to change and evolution as IBM Client Engineering co-creates this solution with our customer.

### Purpose
The purpose of this document is to provide a step-by-step guide for deploying an application on a Satellite location on IBM and AWS using IBM Cloud Console, Terraform, and Helm charts. The solution aims to address the business need to host a business application on different cloud environments to provide a scalable, secure, and efficient process to deploy and run applications.

### Objectives
• Understand Legend app documentation
• Learn to install the Legend app on OpenShift
• Set up Satellite location on IBM Cloud and AWS
• Deploy the Legend app to Satellite locations

### Target Outcomes & Success Metrics
• Automation scripts for deploying Legend to Satellite
• Increased Legend adoption at Goldman Sachs and other shops

### Scope & Roadmap
• Phase 1: Deploy Legend to IBM Cloud and AWS Satellite locations
• Phase 2: Work with F5 SME to add on individual locations and expose individual apps
